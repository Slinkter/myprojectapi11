# SCRUM Process & Workflow

## 1. Roles
*   **Product Owner:** Defines features and acceptance criteria.
*   **Scrum Master:** Remove impediments (Technical Blocks).
*   **Dev Team:** Implementation, Testing, Documentation.

## 2. Artifacts

### Product Backlog
The definitive list of work. Prioritized by business value.

### Sprint Backlog
The subset of items selected for the current iteration.

### Definition of Done (DoD)
A feature is "Done" when:
1.  Code is written and formatted.
2.  Lint checks pass (0 warnings).
3.  Unit tests (if applicable) pass.
4.  Feature meets all Acceptance Criteria (FR & NFR).
5.  Code is documented (JSDoc + Markdown if needed).
6.  Merged to `main` without conflicts.

## 3. Events

### Sprint Planning
*   Select items from Product Backlog.
*   Estimate complexity.
*   Define Sprint Goal.

### Daily Stand-up
*   What did I do yesterday?
*   What will I do today?
*   Any blockers?

### Sprint Review & Demo
*   Showcase working software to stakeholders.

### Sprint Retrospective
*   What went well?
*   What can be improved?

## 4. Issue Tracking (Simulated)
*   **TODO:** Items ready to be worked on.
*   **IN PROGRESS:** Currently being coded.
*   **REVIEW:** Pull Request open, awaiting code review.
*   **DONE:** Merged and deployed.
